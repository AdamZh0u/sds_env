Base Utilities

Author

Jon Reades

Published

September 9, 2022

In order to access the majority of the features that this module uses, you will need to install several ‘base’ utilities:

  • MacOS
  • Windows

You will need to install the following:

  1. The Xcode Command Line Tools, which can usually be installed directly from the Terminal. If that does not work for you, then you can use the Apple Developer website to download and install manually. Note: this may require you to get a free developer account.
  2. Although not strictly necessary, you’ll eventually want the Homebrew package manager, which can also be installed directly from the Terminal.

You will need to install the following:

  1. WSL 2, for which you can follow these instructions.

Once WSL is installed, you will need to familiarise yourself with how having two operating systems (that can talk to each other) works. This will actually be useful for understanding how Docker works, but can be profoundly confusing.

The first, and perhaps most important, thing is understanding where ‘data’ is being stored:

  • Under Linux the user directory is apparently: \\wsl$\<DistroName>\home\<UserName>\
  • Under Windows the user directory is: C:\Users\<UserName>\ or /mnt/c/Users/<UserName>/

These are different locations on your computer’s hard drive, so you will not see your Linux files in your Windows Home Directory and vice versa. To make it easy to switch between the two, I found this page on Microsoft’s web site that goes through some of the post-WSL installation steps, and see especially the additional page on Windows Terminal configuration.

Slightly confusingly, you can run Linux commands directly from Windows, usually be adding wsl in front of the Linux command (e.g. wsl ls *.csv from Windows, where in Linux the command would be ls *.csv).

Common installation errors are covered here.
